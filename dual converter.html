Design and implementation of dual converter by using microcontroller with dc motor as a load



Figure 1: Circuit diagram of the Project


Project objective:


         A microcontroller-based system provides flexibility for the installation of closed loop operation by raising or reducing the firing angle for the dual convertor, allowing the speed of the DC motor to be changed regardless of supply fluctuation and load variation. In twin converters with non-circulating current, one converter operates at a time, while the other is momentarily blocked from conducting by delaying firing pulses to the thyristors. Because only one converter runs at a time, reactors between them are unnecessary. The document provides instructions on how to create regulated power supply and zero crossing detectors for estimating the firing angle using converters. Control circuits are designed to read certain input parameters.


Project Introduction:


         Using integrated circuits, microprocessors, microcontrollers, and embedded chips in electrical and electronic applications minimizes their size, cost, and complexity. Because microcontrollers are programmable, a single circuit may perform many purposes by merely altering the instructions rather than the hardware. This has resulted in widespread use in research, technology, and industry. AC-to-DC conversion is prevalent in power electronics since most systems are AC-powered, yet DC drives are still employed for their easy speed and direction control. A dual converter is used to regulate the speed and direction of DC motors precisely. A dual converter is made up of two complete bridge converters that are connected back to back, one for forward and one for reverse operation.Unlike single-phase full converters, which can only operate in two directions, dual converters can operate in four directions, making them ideal for high-power variable-speed drives. The positive current converter handles forward current, whereas the negative converter handles reverse current.

    ★  Supply pin of port-1 is connected to the negative terminal of the driver circuit.

    ★  The port-2 pins (12,13,14,15,16) of the micro controller is connected to the control switches. These switches give the flexibility to increase and decrease the speed as well as reverse the speed.

    ★  Port-3 pins are connected to the mosfet circuit. Gate circuit of the mosfet is triggered by the opto-couplers.

    ★  Port-3 pins (25,26) are connected to the driver circuit Enable1, Enable2 pins.

    ★  Mosfet terminal point is connected to the ground pin of the driver circuit. The output of the motor driver is connected to the DC gear motor.

Pin diagram of microcontroller:



    ★ 1–8:Port 1: Each of these pins can be used as either input or output. Also, pins 1 and 2 (P1.0 and P1.1) have special functions associated with Timer 2.

    ★ 9:Reset Signal:  High logical state on this input halts the MCU and clears all the registers. positive voltage impulse on this pin resets the MCU. Depending on the device's purpose and environs,
    this pin is usually connected to the push-button, reset-upon-start circuit or a brown out reset circuit.

    ★ 10-17:Port 3:  As with Port 1, each of these pins can be used as universal input or output.

    ★ Pin 10:RXD:  Serial input for asynchronous communication or serial output for synchronous communication.

    ★ Pin 11:TXD:  Serial output for asynchronous communication or clock output for synchronous communication.

    ★ Pin 12:INT0:  Input for interrupt 0.

    ★ Pin 13: INT1:  Input for interrupt 1.

    ★ Pin 14:T0  Clock input of counter 0.

    ★ Pin 15:T1  Clock input of counter 1.

    ★ Pin 16:WR  Signal for writing to external (add-on) RAM memory.

    ★ Pin 17:RD  Signal for reading from external RAM memory.

    ★ 18-19:X2 and X1:  Input and output of internal oscillator. Quartz crystal controlling the frequency commonly connects to these pins.

    ★ 21- 28:Port 2:  If external memory is not present, pins of Port 2 act as universal input/output. If external memory is present, then these pins serve as the location of the higher address byte, i.e. addresses A8 – A15.

    ★ 29:PSEN:  MCU activates this bit (brings to low state) upon each reading of byte (instruction) from program memory. If external ROM is used for storing the program, PSEN is directly connected to its control pins.

    ★ 30:ALE:  Before each reading of the external memory, MCU sends the lower byte of the address register (addresses A0 – A7) to port P0 and activates the output ALE.During the second part of the mechanical MCU cycle, signal on ALE is off, and port P0 is used as Data Bus.

    ★ 31:EA:  Bringing this pin to the logical state zero designates the ports P2 and P3 for transferring addresses regardless of the presence of the internal memory.

    ★ 32-39:Port0:  Similar to Port 2, pins of Port 0 can be used as universal input/output, if external memory is not used. If external memory is used, P0 behaves as address output (A0 – A7) when ALE pin is at high logical level, or as data output (Data Bus) when ALE pin is at low logical level.

    ★ 40:VCC:  Power +5V
