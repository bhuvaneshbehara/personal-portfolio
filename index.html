Input files for PD

Input files play a crucial part in the development of chips. There are two kinds of files in PD flow. They are technology and design-related files.

Read More

  • Design Related files
    1. Gate-level netlist (.v)
    2. Constraints file (.sdc)
    3. Power intent file (.upf)
    4. Scandef (.scandef)
    5. GDSII file (.gds)
  • Technology related files
    1. Logical Libraries (.lib or .db)
    2. Physical Libraries (.LEF)
    3. Technology file (.tf)
    4. Tluplus file(.tlu+)
    5. Map file (.map)

  • About Design Related Files

    1. Gate-level netlist(.V)

      2.  This file includes verilog code that makes logic connections between cells. It also contains instances of standard cells, module port names, macro pin connections, and network connections. It belongs to the structural or gate-level model.

    2. Constraints file(.sdc)

      3.  SDC is a short form of synopsys design Constraints. It is in the format of IEEE 1801 format. It contains clock definitions, timing exceptions, design rule constraints(like max_fanout, max_trans),operating conditions,timing constraints.

    3. Power intent file(.upf)

      4. The acronym UPF stands for unified power format. It includes information about the power domain, power supply, power nets, power switches, and low-power approaches such as level shifter placement, isolation cell position, and power state table. It also includes level shifters, isolation cells, retention registers, and always-on buffers.

    4. Scandef (.scandef):

      5. It carries scan-chain information. The scandef was created by the DFT team. It provides scan cell reordering limitations as well as scan enable flops for clocks..

    5. GDSII file(.gds):

      6.   The abbreviation "GDS" stands for graphic data system. It comprises information on physical and time constraints for the complete design that is ready for tapeout. This GDSII file is sent to the foundry, which manufactures the chip depending on the information in it.

About Technology related files

Technology files are important in PD. This technology file includes design rules, information on metal, and more. These files help to close the gap between the foundary and design.

Read More

  • Technology related files
    1. Logical Libraries(.lib or .db):

        Lib file is a short form of liberty timing file.It is a human-redable format. It contains cell related information like cell name, PG pin name, Area of cell, Leakage power, pin details etc. It also contain timing related information like Setup, hold, timing arcs, transition, wire-load models,Look-up tables for timing and power calculation.

    2. Physical Libraries (.LEF):

        The LEF file is the abstract layout information of std cells. It contains information related to geometry of each cell like cell size, cell name, symmetry of cell,pin information (like pin name, direction of pin). It also contain antenna gate information and rectangular coordinates for metal layer.

    3. Technology File(.tf):

        It contains the information about metal layers, vias and their design rules.It contains information like color and stipple details of metal layers,tile height and width of the std cells,Pitch of metal layer,minwidth of metal layer,blink,pattern,metal layer name,via contactcode,via cut spacing rules,design rules(like windowSize,mindensity,maxdensity,minEnclosure) for metal layers.

    4. Tluplus(.tlu+):

      It contains RC parasitics information to calculate the net delay. Based on different algorithms this file will generate.This file used at placement stage.

    5. Map File(.map):

      It used for mapping the files. For example, the ITF file has the name metal1, but because tlu+ only has M1 at the moment, we utilized this map file.

Macro Guidelines for Floorplanning

Macro placement is a crucial step in PD. Read More

  • Group the macros by hierarchy.
    •  Why: If you place macros not by same group hierarchy then wirelength increases which leads to timing violations. And Module Splitting happen which leads to global congestion.
  • Place the macros based on flylines.
    •  Why:The wirelength will be reduced if you position macros according to flylines (since you will know the wire length).Congestion reduction will occur. And to improve the timing closure.
  • Give the macro channel.
    •  Why:If you provide a space between macros, buffers or inverters are added for optimization or timing correction. Furthermore, to minimize macro pin congestion and to make macro pin routing easier. In addition, power-related violations, such as power straps, will travel over the channel.
  • Avoid stacking.
    •  Why: If you avoid stacking macros, you have more routing resources, which means less routing congestion.
  • Avoid notches.
    •  Why: There is still space if you are not avoiding notches. This gap necessitates an additional mask during manufacture, which results in additional expenses.
  • Avoid Criss-cross.
    •  Why: If you connecting macros using the Criss-cross approach, the tool will automatically redirect the routing net to another path with a larger length, resulting in routing congestion and timing concerns.
  • Avoid blocking of ports.
    •  Why: If you block ports, you'll need extra routing resources to connect buffers or inverters if there are soft blockages between them.
  • Apply keep-out margin.
    •  Why:If you add a keep-out margin, the macro pin will be free, resulting in no short circuit. Also useful for power-related violations, such as avoiding the breakdown of macro ring metal layers.
  • Apply routing blockages over the macro.
    •  Why: If you place a routing blockage over the macros, no short circuits occur.

    Conclusion

  • By following above guidelines you can achieve less congestion, optimized routing, and good for further stages.
